Audit Check limit-setinfo-size

VMware has the ability to add custom “name = value” variables into the VMX Configuration files of the individual Guest Virtual Machines. By default, the limit on this is 1MB, which makes for an extremely large configuration file. An indirect DOS attack could be mounted, increasing all the VMX files by this maximum value, which would dramatically increase the startup time for each guest.

To limit the amount of custom information, the following VMX parameters are recommended: (this is the default value, a smaller value is generally recommended, to be determined by the policy and requirements of individual VMs):
Tools.setInfo.sizeLimit=1048567

Audit Check: 
vmrun -T esx -h https://<host ip>/sdk -u <userid> -p <password> readvariable <path to vmx file> Tools.setInfo.sizeLimit
(note that this can only check running VMs, use vmware-cmd for dormant hosts)

PowerCLI:
    $vm | Get-AdvancedSetting -Name  "tools.setInfo.sizeLimit"

Tools.setInfo.sizeLimit=1048567 is Compliant, other values or -1 (blank) is Non-Compliant
This value will of course vary from organisation to organisation.
