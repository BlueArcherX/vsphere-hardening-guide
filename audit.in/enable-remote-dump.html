Audit Check enable-remote-dump

Configure a centralized location to collect ESXi host core dumps using the "ESXi Dump Collector" When a host crashes, an analysis of the resulting core dump can be helpful in determining the cause of the crash and in working with VMware support to resolve the issue. Using a centralized dump collector helps ensure that core dump files are available for support situations of this type. Audit Check:

	In vCLI:
		esxcli system coredump network get
        In PowerCLI:
                $esxcli = Get-EsxCli -VMHost $VMHost
                $esxcli.system.coredump.network.get()
