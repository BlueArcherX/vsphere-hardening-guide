Audit Check check-privilege-reassignment

When VMware vCenter Server restarts, the log file should be scanned to ensure that no privileges were reassigned by the restart.

To assess this, check the log vpxd.log after reboot of vCenter, looking for privilege reassignment.

Audit Check: 
psexec \\<VCENTERSERVERNAME> -u <userid> -p <password> type "%ALLUSERSPROFILE%\VMware\VMware VirtualCenter\Logs\vpxd.log" | find /i "assigned"
(this assumes that vCenter is on a Windows Server 2008 Host)

or in PowerCLI:
    Get-VIPrivilege -server $vc | select server,name, privilegelist
