Audit Check install-with-service-account

The vCenter application by default installs with “LocalSystem” privileges. In reality, less rights are required for this service to run. vCenter should be modified after installation to use a named account, with less rights (exact rights are outlined in the VMware’s installation guides).

This has the added advantage that if a back end SQL server is used for vCenter on another host, this named account can then access SQL using Windows User Authentication, and can be granted the specific required SQL permissions.

Audit Check: 
    wmic /node:vcenterserver service get DisplayName, StartName  | find /i “vmware"
or in PowerShell: get-wmiobject win32_service -computer $vc.name -credential $vicreds | Where-Object { $_.displayname -like "*VMware*" } Audit Results:
