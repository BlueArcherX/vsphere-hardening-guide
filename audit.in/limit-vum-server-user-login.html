Audit Check limit-vum-server-user-login

Because of the power available within the VUM interface, it’s a good idea to log its use, to ensure that only valid personnel are accessing it, at the correct times. It’s also good to track failed attempts to login, which might catch penetration attempts and attacks.

To audit login events (to verify good behaviour or bad), a good starting point is to audit the following Windows Events:

Event ID 675 – incorrect password Event ID 676 – failed login attempt for reasons other than incorrect password (mistyped username, account locked etc) Event ID 680 - Successful login Event ID 4624 - Successful login Event ID 4672 - Special Privileges Assigned Event ID 4625 - Failed Login Attempt

These events also log the station that originated the request

Audit Check:

While these events can be parsed using WMIC NTEVENT, this method is very CPU intensive

PSLOGLIST (part of the Microsoft Systinternal Suite) gives us a much better approach to this. To extract occurrences of these three events for instance, you might use a command string like: 
	psloglist \\<VCENTERSERVERNAME> -u <userid> -p <password> –s –I 676,675,680 Security
