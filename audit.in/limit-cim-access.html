Audit Check limit-cim-access

The CIM API interface allows access to lower level, hardware components in the system. Because of this low-level access, it’s recommended that an account with restricted access be created for CIM access (any local account has CIM access)

Audit Check:

For a manual assessment of user privileges, based on CIM management requirements

If CIM write access is required, 2 rights are required:

Host / Config / SystemManagement , and Host / CIM / CIMInteraction

In Powershell:
    get-vmhostaccount -server $esxh.name

NOTE: This check only works when run directly against an ESXi host (not against a vCenter host)
