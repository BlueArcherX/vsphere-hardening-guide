Audit Check disable-autoinstall - Disable VMware Tools Auto-Install

VMware Tools auto install can initiate an automatic reboot, disabling this option will prevent tools from being installed automatically and prevent automatic machine reboots.

It is also desirable from a security perspective to test updates before deploying in a production VM.

The affected VMX settings are:
Isolation.tools.autoinstall.disable=TRUE

Audit Check:

    vmrun -T esx -h https://<host ip>/sdk -u <userid> -p <password> readvariable <path to vmx file> Isolation.tools.autoInstall.disable
    (note that this method can only check running VMs, use vmware-cmd for dormant hosts)

In PowerCLI:
    $vm | Get-AdvancedSetting -Name  "isolation.tools.autoInstall.disable"

A returned value of "TRUE" or blank indicates Compliance, "False" indicates non-compliance
