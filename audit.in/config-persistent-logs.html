Audit Check config-persistent-logs 

Local logging within ESXi is only held for one day, and is not persistent across reboots.  This is further complicated by the fact that most of
the local filesystem is non-persistent, the only persistent directory being /scratch.

For this reason, if local logging is required within ESXi, it’s recommended to direct it to a NFS or VMFS volume.

Audit Check:
	For ESXi 4:
		vicfg-syslog –server <host ip> --user <userid> --password <password>  --show
	For ESXi 5:
		Esxcli --server <hostip> --username <user> --password <password> system syslog config get
        In PowerCLI:
                $esxhost | Get-AdvancedSetting Syslog.global.logDir
