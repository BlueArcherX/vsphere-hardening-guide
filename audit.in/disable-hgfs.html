Audit Check disable-hgfs

Certain automated operations such as automated tools upgrades use a component in the hypervisor called "Host Guest File System" and an attacker could potentially use this to transfer files inside, into or out of the guest OS

The affected VMX setting is:
isolation.tools.hgfsServerSet.disable=TRUE

(note that this setting is used by VMware Tools auto-update, so is only desired in exceptional circumstances)

Audit Check: 

vmrun -T esx -h https://<host ip>/sdk -u <userid> -p <password> readvariable <path to vmx file> Isolation.tools.hgfsServerSet.disable
(note that this can only check running VMs, use vmware-cmd for dormant hosts)

In PowerCLI:
    $vm | Get-AdvancedSetting -Name  "isolation.tools.hgfsServerSet.disable"

A value of "TRUE" or blank indicates Compliance, "False" indicates non-compliance
