Audit Check enable-nfc-ssl

On the face of it, enabling SSL for vMotion and other host copy operations would be considered desirable, but this recommendation has a few issues. First of all, encrypting large volumes of data can take a measurable amount of CPU resources. More importantly, this feature is not yet fully tested by VMware. Since copies of this type are only supposed to happen over the management network, this measure is only recommended in special circumstances. This setting is controlled by the "usessl" parameter in vCenter's VPXD.CFG, which is an XML formated configuration file.

Powershell provides a native method of extracting XML information out VPXD.CFG. PSTools provides a method of executing the required powershell command on the remote host (reference http://isc.sans.edu/diary.html?storyid=11992 for a more complete explanation of this operation)

Audit Check:
psexec \\<vcenter host> -u <Username> -p <Password>  powershell -Command "[xml]$vpxd = Get-Content %ALLUSERPROFILE%\Application Data\VMware\VMware VirtualCenter\vpxd.cfg" ; "write-Host $vpxd.config.useSSL"
